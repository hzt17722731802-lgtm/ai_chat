## 概述

本文档描述了客服端系统的所有 REST API 接口，包括客服登录、绩效管理、AI 对话和聊天记录管理等功能。

**基础路径**: `/`

**认证方式**: JWT Token（除登录接口外，其他接口均需在请求头中携带）

**请求头格式**: 
```
Authorization: Bearer <token>
```

---

## 统一返回格式

所有接口均使用统一的返回格式：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {}
}
```

**字段说明**:

| 字段 | 类型 | 说明 |
|------|------|------|
| code | Integer | 状态码，200 表示成功，500 表示失败 |
| message | String | 响应消息 |
| data | Object | 响应数据，具体结构因接口而异 |

## 认证接口

### 客服登录

客服人员进行系统登录。

**接口地址**: `POST /service/login`

**请求格式**: `application/json`

**请求参数**:

```json
{
  "employeeNo": "客服工号",
  "password": "密码"
}
```

**请求参数说明**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| employeeNo | String | 是 | 客服工号 |
| password | String | 是 | 登录密码 |

**响应示例**:

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "employeeId": "员工 ID",
    "employeeNo": "客服工号",
    "name": "姓名",
    "department": "部门",
    "position": "职位"
  }
}
```

**响应数据说明**:

| 字段 | 类型 | 说明 |
|------|------|------|
| token | String | JWT 令牌，后续请求需携带此令牌 |
| employeeId | String | 员工 ID |
| employeeNo | String | 客服工号 |
| name | String | 姓名 |
| department | String | 部门 |
| position | String | 职位 |

**错误码**:

| 状态码 | 说明 |
|--------|------|
| 200 | 登录成功 |
| 500 | 员工不存在/密码错误/账号已被禁用/登录失败 |

### 获取当前用户信息

获取当前登录客服的详细信息。

**接口地址**: `GET /service/current`

**认证**: 需要 JWT Token

**请求头**:

```
Authorization: Bearer <token>
```

**响应示例**:

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "id": "员工 ID",
    "employeeNo": "客服工号",
    "name": "姓名",
    "department": "部门",
    "position": "职位",
    "phone": "手机号",
    "email": "邮箱",
    "performanceScore": 95.5,
    "salesAmount": 100000.00,
    "completedOrders": 150,
    "status": "ACTIVE",
    "createTime": "2024-01-01T00:00:00",
    "updateTime": "2024-01-01T00:00:00"
  }
}
```

**响应数据说明**:

| 字段 | 类型 | 说明 |
|------|------|------|
| id | String | 员工 ID |
| employeeNo | String | 客服工号 |
| name | String | 姓名 |
| department | String | 部门 |
| position | String | 职位 |
| phone | String | 手机号 |
| email | String | 邮箱 |
| performanceScore | BigDecimal | 绩效分数 |
| salesAmount | BigDecimal | 销售金额 |
| completedOrders | Integer | 完成订单数 |
| status | String | 状态（ACTIVE: 正常） |
| createTime | LocalDateTime | 创建时间 |
| updateTime | LocalDateTime | 更新时间 |

**错误码**:

| 状态码 | 说明 |
|--------|------|
| 200 | 获取成功 |
| 500 | 无效的令牌/用户不存在/获取用户信息失败 |

## 绩效接口

### 获取绩效数据

获取当前登录客服的绩效数据，包括绩效分数、销售金额、完成订单数和平均评分。

**接口地址**: `GET /service/performance`

**认证**: 需要 JWT Token

**请求头**:

```
Authorization: Bearer <token>
```

**响应示例**:

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "employeeId": "员工 ID",
    "employeeNo": "客服工号",
    "name": "姓名",
    "department": "部门",
    "position": "职位",
    "performanceScore": 95.5,
    "salesAmount": 100000.00,
    "completedOrders": 150,
    "averageRating": 4.8
  }
}
```

**响应数据说明**:

| 字段 | 类型 | 说明 |
|------|------|------|
| employeeId | String | 员工 ID |
| employeeNo | String | 客服工号 |
| name | String | 姓名 |
| department | String | 部门 |
| position | String | 职位 |
| performanceScore | BigDecimal | 绩效分数 |
| salesAmount | BigDecimal | 销售金额 |
| completedOrders | Integer | 完成订单数 |
| averageRating | BigDecimal | 平均评分（从评分表计算得出） |

**错误码**:

| 状态码 | 说明 |
|--------|------|
| 200 | 获取成功 |
| 500 | 无效的令牌/用户不存在/获取绩效数据失败 |

## 附录

### 错误码汇总

| 状态码 | 说明 |
|--------|------|
| 200 | 操作成功 |
| 500 | 操作失败（具体原因见 message 字段） |

### 常见错误信息

- `员工不存在`: 登录时提供的工号不存在
- `密码错误`: 登录密码错误
- `账号已被禁用`: 员工账号状态不是 ACTIVE
- `无效的令牌`: JWT Token 无效或已过期
- `用户不存在`: Token 中的用户 ID 在数据库中不存在

### 技术栈

- **后端框架**: Spring Boot
- **AI 框架**: Spring AI
- **数据库**: MySQL + Neo4j
- **缓存**: Redis
- **认证**: JWT
- **AI 模型**: Ollama (medic_qwen0.6b)
